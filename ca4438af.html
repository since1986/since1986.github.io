正文

设计理念

  • 尽最大努力尽量避免坏情况发生：在 hash() 中用扰动函数力图将 hash 冲突降至最低
  • 当坏情况不可避免地发生了，则尽量降低其带来的负面影响：hash 冲突产生了用槽位存储普通链表或红黑树型链表解决
  • 特事特办：在 get 时先尝试匹配首节点，如果匹配直接返回，如果不匹配才走标准查找算法
  • 快速失败

基本逻辑

  • 利用数组 table 存储 Node，每个 table 的槽位中的 Node 有可能是普通单向链表或者红黑树链表
  • put(key, value) 时，对 key 求 hash 值，把求出的 hash & (capacity - 1) 得出在 table 中的 index，当 index 对应table 槽位上无现有 Node 则直接用 key、value 创建 Node，若已有 Node存在（也就是hash冲突）则判断key是否相等，相等则更新，不相等则创建Node的后继节点，在创建后继节点时如果链长度到了树化阈值则将Node转化为红黑树链表；同时，如果 table 实际已占用槽位总数 > capacity * loadFactor 则进行扩容
